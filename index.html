Security hardening

Threat model and mitigations for NeuroRadReporter v0.9.0-pre. All issues listed below are fixed in this build.

XSS — DICOM metadata injection
Severity: High — Fixed
  • CVE-classDICOM string tags (Series Description, Modality, UID, Body Part, hash) were interpolated raw into innerHTML. A malicious DICOM file with <script> in any string field executed arbitrary JS.
  • FixAll DICOM strings pass through sanitizeText() before DOM insertion. Table rows and series list items are built with textContent assignments, never innerHTML string concatenation.
XSS — AI report output injection
Severity: High — Fixed
  • CVE-classLLM output was passed to fmtRep() which injected text into innerHTML. A prompt-injected or adversarially crafted report could execute JS in the user's browser.
  • FixfmtRep() now creates DOM nodes and assigns via textContent only. No AI-generated content ever touches innerHTML.
onclick attribute injection via Series UID
Severity: Medium — Fixed
  • INJSeries UIDs were embedded in onclick="openRep('UID')" strings. A UID containing ') breaks the attribute boundary and executes arbitrary JS.
  • FixAll onclick attributes removed. Navigation uses data-uid attributes + addEventListener. Event delegation handles all dynamic elements.
Prototype pollution via DICOM UID
Severity: Medium — Fixed
  • PROTOSM[uid] = {...} used a plain object keyed by DICOM UID. A crafted UID of __proto__ or constructor polluted Object.prototype for the entire page.
  • FixSeries state uses new Map() — Maps have no prototype chain exposure, so arbitrary string keys are safe.
Subresource integrity (SRI)
Severity: Low — Fixed
  • SRIdicom-parser loaded from jsDelivr CDN without an integrity= hash. A CDN compromise would silently deliver malicious code with full DOM and memory access.
  • Fixintegrity="sha384-..." and crossorigin="anonymous" added to the script tag. Browser rejects the script if the hash doesn't match.
Content Security Policy
Severity: Low — Fixed
  • CSPNo CSP meant any injected script could exfiltrate data (API key, patient hashes, report content) to arbitrary external origins.
  • FixCSP meta tag: connect-src limited to api.anthropic.com only. object-src none, base-uri self. Blocks data exfiltration to unknown origins.
File size / memory DoS
Severity: Medium — Fixed
  • DoSNo limit on file count or size. Dropping thousands of large DICOM files exhausted browser memory and crashed the tab.
  • FixHard limits enforced at import: max 200 files per batch, max 200 MB per file. Excess files are rejected with a clear error message.
API key scope
Severity: Low — Mitigated
  • INFOAPI key was a global let variable, readable from the browser console by any injected script.
  • FixKey is now scoped inside an IIFE — not accessible as a global. Residual risk: key remains readable from browser memory inspector (inherent to client-side use — document in threat model and require key rotation after sessions).
